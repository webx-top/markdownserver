两个队列采󲙢了不同的设计。本地队列󵰓度固定，󲙢数组󳯩然是效率最󶒠。󳣐全局队列󵰓
度未知，只能󲙢链表实现。
调度器在很多地󰸵都采󲙢两级队列设计，本地队列是为了当前线程󰺂锁获取资源，󳣐全局
队列则是为了在多个 P/M 间进󴘲平衡。当 P 管理的对象数量过多时就会上交部分到全
局，反过来，就从全局提取批到本地。总之，最终󲥣的是为了更好地复󲙢内存，更快地
完成任务执󴘲。
3.3 任务线程
不管语󴥧层󵼃如何抽象，所有 G 任务总归要由线程执󴘲，每个系统线程对应个 M。
runtime.h
struct M
{
G* g0; // 运󴘲时管理栈。
void (*mstartfn)(void); // 启动函数，󱞳如执󴘲 sysmon。
G* curg; // 当前运󴘲的 G。

先了解 M 的创建过程。
proc.c
static void newm(void(*fn)(void), P *p)
{

mp = runtime·allocm(p);
// 设置待绑定 P 和启动函数。
mp->nextp = p;
mp->mstartfn = fn;
// 创建系统线程。
runtime·newosproc(mp, (byte*)mp->g0->stack.hi);
}
