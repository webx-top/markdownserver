func forcegchelper() {
forcegc.g = getg()
forcegc.g.issystem = true
for {
// 休眠该 goroutine。
// park 会暂停 goroutine，但不会放回待运󴘲队列。
goparkunlock(&forcegc.lock, "force gc (idle)")
// 唤醒后，执󴘲强制垃圾回收。
gogc(1)
}
}
这个 forcegc.g 会循环执󴘲，每次完成后休眠，直到被 sysmon 重新返回任务队列。

值就是 2GB，这可能导致很󵰓时间󰺂法触发回收操作。这就存在很的内存浪费，所以强
制回收是󵻸常必要的。
